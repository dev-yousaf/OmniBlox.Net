Multi-Tenant Global Query Filters & the Products CRUD

Every OmniBlox request belongs to a company (tenant). In this lesson we'll build a global query filter that automatically scopes every database query to the current company — no manual .Where(p => p.CompanyId == ...) required. Then we'll prove it works by building a complete Products CRUD — the first business feature in the app — with Create, Read (list + detail), Update, and Delete endpoints all behind the multi-tenant filter.

The frontend (Next.js) already expects products at /products with fields like name, sku, salePrice, costPrice, stock, category, unit, and many more. We'll build the API to match.

What changes: 1 new interface, 2 new enums, 1 new entity, 1 EF config, 5 new files in Application (commands + queries + DTOs), 1 controller, 1 migration, and 3 existing files modified. No new NuGet packages — MediatR auto-discovers our new handlers.

The Big Picture — Multi-Tenant Architecture

How every request is automatically scoped to one company
When a user authenticates, the JWT contains a companyId claim. The CurrentUserService reads this claim and exposes it as CompanyId. When AppDbContext builds its model, it inspects every entity type: if the entity implements ITenantEntity (which has a CompanyId property), it attaches a global query filter: e => e.CompanyId == currentCompanyId. Every LINQ query — ToListAsync, FirstOrDefaultAsync, CountAsync — automatically includes this WHERE clause. The developer never writes .Where(p => p.CompanyId == id). Even .Include() and navigation properties get filtered.

When a new entity is added, the overridden SaveChangesAsync checks if the entity's CompanyId is still Guid.Empty (the default for new entities) and fills it from the current user's company. The developer never sets CompanyId manually.
File chain: ITenantEntity.cs → AppDbContext.cs → CurrentUserService.cs → (any handler that queries Products)

Part 1 — Multi-Tenant Foundation

1. ITenantEntity — the contract

src/OmniBlox.Domain/Interfaces/ITenantEntity.cs
namespace OmniBlox.Domain.Interfaces;

public interface ITenantEntity
{
    Guid CompanyId { get; set; }
}
Minimal interface: One property. Any entity that belongs to a company (Products, Customers, Invoices, Warehouses, etc.) implements this. The CompanyId is both the FK to the Companies table and the discriminator for the query filter.
Why an interface instead of a base class?
ITenantEntity is an interface because entities already inherit from BaseEntity. C# doesn't allow multiple inheritance, so an interface is the only way to add tenant behavior across the hierarchy. The AppDbContext uses typeof(ITenantEntity).IsAssignableFrom(entityType.ClrType) to detect which entities need the query filter — it works with any class that implements the interface, regardless of its place in the inheritance tree.
File chain: ITenantEntity.cs → AppDbContext.cs → Product.cs

2. CurrentUserService — no more throwing on anonymous requests

Previously, CompanyId threw UnauthorizedAccessException when the JWT was missing. But AppDbContext is created for every request (including anonymous signup/login), so the query filter would crash. Now it returns Guid.Empty instead — handlers that require auth already have [Authorize] protection.

src/OmniBlox.Infrastructure/Auth/CurrentUserService.cs (changed lines)
    public Guid CompanyId
    {
        get
        {
            var value = _httpContextAccessor.HttpContext?.User
                .FindFirstValue("companyId");

            return value is not null ? Guid.Parse(value) : Guid.Empty;
        }
    }
Why this is safe: Guid.Empty means "no company". If an anonymous request somehow queries a tenant-scoped entity, the filter becomes e => e.CompanyId == Guid.Empty, which returns zero rows — no data leaked. Authenticated endpoints get the real CompanyId from the JWT because the token was validated before the handler runs.

3. AppDbContext — query filters + auto-set CompanyId

Three changes in one file:

src/OmniBlox.Infrastructure/Persistence/AppDbContext.cs (full file)
using System.Linq.Expressions;
using Microsoft.EntityFrameworkCore;
using Microsoft.EntityFrameworkCore.ChangeTracking;
using OmniBlox.Application.Common.Interfaces;
using OmniBlox.Domain.Entities;
using OmniBlox.Domain.Interfaces;

namespace OmniBlox.Infrastructure.Persistence;

public class AppDbContext : DbContext, IApplicationDbContext
{
    private readonly ICurrentUserService _currentUser;

    public AppDbContext(
        DbContextOptions<AppDbContext> options,
        ICurrentUserService currentUser)
        : base(options)
    {
        _currentUser = currentUser;
    }

    public DbSet<Company> Companies => Set<Company>();
    public DbSet<User> Users => Set<User>();
    public DbSet<Product> Products => Set<Product>();

    protected override void OnModelCreating(ModelBuilder builder)
    {
        base.OnModelCreating(builder);
        builder.ApplyConfigurationsFromAssembly(typeof(AppDbContext).Assembly);

        foreach (var entityType in builder.Model.GetEntityTypes())
        {
            if (typeof(ITenantEntity).IsAssignableFrom(entityType.ClrType))
            {
                var parameter = Expression.Parameter(entityType.ClrType, "e");
                var property = Expression.Property(parameter, nameof(ITenantEntity.CompanyId));
                var companyId = Expression.Constant(_currentUser.CompanyId);
                var body = Expression.Equal(property, companyId);
                var lambda = Expression.Lambda(body, parameter);

                entityType.SetQueryFilter(lambda);
            }
        }
    }

    public override async Task<int> SaveChangesAsync(CancellationToken ct = default)
    {
        foreach (var entry in ChangeTracker.Entries<ITenantEntity>())
        {
            if (entry.State == EntityState.Added && entry.Entity.CompanyId == Guid.Empty)
            {
                entry.Entity.CompanyId = _currentUser.CompanyId;
            }
        }

        return await base.SaveChangesAsync(ct);
    }
}
Syntax breakdown — Expression Trees:
The HasQueryFilter method accepts a Lambda expression. Instead of writing e => e.CompanyId == someValue at compile time (we don't know which entity types exist), we build the expression at runtime:
Expression.Parameter(entityType.ClrType, "e") — creates the e parameter
Expression.Property(parameter, "CompanyId") — creates e.CompanyId
Expression.Constant(_currentUser.CompanyId) — the current value from DI
Expression.Equal(property, companyId) — creates e.CompanyId == value
Expression.Lambda(body, parameter) — wraps it in e => ...
entityType.SetQueryFilter(lambda) — attaches it as the global filter.
Important — User does NOT implement ITenantEntity: Login needs to look up users by email across all companies. If we filtered Users, login would only find users in the Guid.Empty company (i.e., none). User access is controlled manually via JWT claims (_currentUser.UserId) rather than a global filter.
How the query filter is evaluated at runtime
OnModelCreating runs once per application lifetime (the model is cached). But the expression tree it creates captures a reference to the _currentUser field of the AppDbContext instance. Since each HTTP request gets its own scoped AppDbContext (and therefore its own ICurrentUserService), the _currentUser.CompanyId is evaluated per query, not once at startup. When User A from Company X queries products, EF Core translates the filter to WHERE "p"."CompanyId" = 'company-x-guid'. User B from Company Y gets WHERE "p"."CompanyId" = 'company-y-guid' — automatically, from the same compiled model.
File chain: AppDbContext.cs → (model cache) → query pipeline
How SaveChangesAsync auto-sets CompanyId
When a handler creates a new Product with new Product { Name = "Widget", ... }, it doesn't set CompanyId — it stays Guid.Empty. Before the INSERT reaches PostgreSQL, the overridden SaveChangesAsync iterates all tracked entities implementing ITenantEntity, checks if they're in EntityState.Added with CompanyId == Guid.Empty, and fills in the current user's CompanyId. The developer never writes product.CompanyId = _currentUser.CompanyId in any handler. The handler stays clean: just set business fields, save, and the infrastructure handles tenancy.
File chain: AppDbContext.SaveChangesAsync() → Product entity → database

using Microsoft.EntityFrameworkCore;
using Microsoft.EntityFrameworkCore.Metadata.Builders;
using OmniBlox.Domain.Entities;

namespace OmniBlox.Infrastructure.Persistence.Configurations;

public class ProductConfiguration : IEntityTypeConfiguration<Product>

    public void Configure(EntityTypeBuilder<Product> builder)
    {
        builder.ToTable("Products");

        builder.HasKey(p => p.Id);

        builder.Property(p => p.Name)
            .HasMaxLength(200).IsRequired();

        builder.Property(p => p.SKU)
            .HasMaxLength(100).IsRequired();

        builder.Property(p => p.Description)
            .HasMaxLength(2000);

        builder.Property(p => p.Type)
            .HasConversion<int>().IsRequired();

        builder.Property(p => p.Category)
            .HasMaxLength(100).IsRequired();

        builder.Property(p => p.Brand)
            .HasMaxLength(100);

        builder.Property(p => p.Unit)
            .HasMaxLength(50).IsRequired();

        builder.Property(p => p.ImageUrl)
            .HasMaxLength(500);

        builder.Property(p => p.SalePrice)
            .HasColumnType("decimal(18,2)").IsRequired();

        builder.Property(p => p.CostPrice)
            .HasColumnType("decimal(18,2)").IsRequired();

        builder.Property(p => p.Status)
            .HasConversion<int>().IsRequired();

        builder.Property(p => p.BarcodeSymbology)
            .HasMaxLength(50);

        builder.Property(p => p.TaxRate)
            .HasColumnType("decimal(5,2)");

        builder.Property(p => p.ItemCode)
            .HasMaxLength(100);

        builder.Property(p => p.Manufacturer)
            .HasMaxLength(200);

        builder.Property(p => p.Warranty)
            .HasMaxLength(200);

        builder.HasIndex(p => new { p.CompanyId, p.SKU }).IsUnique();

        builder.HasOne(p => p.Company)
            .WithMany()
            .HasForeignKey(p => p.CompanyId)
            .OnDelete(DeleteBehavior.Restrict);
    }
}
Composite unique index: HasIndex(p => new { p.CompanyId, p.SKU }).IsUnique() means SKUs must be unique within a company. Two companies can both have SKU "WIDGET-001" — they're different products in different tenants. Without this index, the multi-tenant filter would still separate them visually, but a bug could create duplicate SKUs silently.

8. CreateProductCommand — command + handler + validator

src/OmniBlox.Application/Features/Products/Commands/CreateProductCommand.cs
using FluentValidation;
using MediatR;
using Microsoft.EntityFrameworkCore;
using OmniBlox.Application.Common.Interfaces;
using OmniBlox.Application.Features.Products.DTOs;
using OmniBlox.Domain.Entities;
using OmniBlox.Domain.Enums;
using OmniBlox.Shared.Exceptions;

namespace OmniBlox.Application.Features.Products.Commands;

public record CreateProductCommand : IRequest<ProductDto>
{
    public string Name { get; init; } = string.Empty;
    public string SKU { get; init; } = string.Empty;
    public string? Description { get; init; }
    public string Type { get; init; } = "STANDARD";
    // ... 17 more fields matching the DTO
    public DateTime? ExpiryDate { get; init; }
}

public class CreateProductCommandHandler
    : IRequestHandler<CreateProductCommand, ProductDto>
{
    private readonly IApplicationDbContext _context;

    public CreateProductCommandHandler(IApplicationDbContext context)
    {
        _context = context;
    }

    public async Task<ProductDto> Handle(
        CreateProductCommand request, CancellationToken ct)
    {
        var skuExists = await _context.Products
            .AnyAsync(p => p.SKU == request.SKU, ct);

        if (skuExists)
            throw new ConflictException(
                $"Product with SKU '{request.SKU}' already exists.");

        var product = new Product
        {
            Name = request.Name,
            SKU = request.SKU,
            Description = request.Description,
            Type = Enum.TryParse<ProductType>(request.Type, true,
                out var type) ? type : ProductType.STANDARD,
            // ... map remaining fields
            ExpiryDate = request.ExpiryDate,
        };

        _context.Products.Add(product);
        await _context.SaveChangesAsync(ct);

        return ProductDto.FromEntity(product);
    }
}

public class CreateProductCommandValidator
    : AbstractValidator<CreateProductCommand>
{
    public CreateProductCommandValidator()
    {
        RuleFor(v => v.Name)
            .NotEmpty().MaximumLength(200);
        RuleFor(v => v.SKU)
            .NotEmpty().MaximumLength(100);
        RuleFor(v => v.SalePrice)
            .GreaterThanOrEqualTo(0);
        RuleFor(v => v.TaxRate)
            .InclusiveBetween(0, 100).When(v => v.TaxRate.HasValue);
        // ... more validation rules
    }
}
No CompanyId in the handler: The handler creates new Product { Name, SKU, ... } — it never touches CompanyId. The SaveChangesAsync override in AppDbContext fills it automatically. If the SKU already exists in any company (but the composite index is per-company), the AnyAsync check is also scoped by the query filter — so it only checks within the current company.

9. GetProductsQuery — paginated list with search/filter

src/OmniBlox.Application/Features/Products/Queries/GetProductsQuery.cs
using MediatR;
using Microsoft.EntityFrameworkCore;
using OmniBlox.Application.Common.Interfaces;
using OmniBlox.Application.Features.Products.DTOs;

namespace OmniBlox.Application.Features.Products.Queries;

public record GetProductsQuery : IRequest<ProductListResponse>
{
    public int Page { get; init; } = 1;
    public int Limit { get; init; } = 20;
    public string? Search { get; init; }
    public string? Category { get; init; }
    public string? Status { get; init; }
}

public class GetProductsQueryHandler
    : IRequestHandler<GetProductsQuery, ProductListResponse>
{
    private readonly IApplicationDbContext _context;

    public GetProductsQueryHandler(IApplicationDbContext context)
    {
        _context = context;
    }

    public async Task<ProductListResponse> Handle(
        GetProductsQuery request, CancellationToken ct)
    {
        var query = _context.Products.AsQueryable();

        if (!string.IsNullOrWhiteSpace(request.Search))
        {
            var search = request.Search.ToLower();
            query = query.Where(p =>
                p.Name.ToLower().Contains(search) ||
                p.SKU.ToLower().Contains(search));
        }

        if (!string.IsNullOrWhiteSpace(request.Category))
            query = query.Where(p => p.Category == request.Category);

        if (!string.IsNullOrWhiteSpace(request.Status)
            && Enum.TryParse<Domain.Enums.ProductStatus>(
                request.Status, true, out var status))
            query = query.Where(p => p.Status == status);

        var total = await query.CountAsync(ct);

        var items = await query
            .OrderByDescending(p => p.CreatedAt)
            .Skip((request.Page - 1) * request.Limit)
            .Take(request.Limit)
            .ToListAsync(ct);

        return new ProductListResponse
        {
            Items = items.Select(ProductDto.FromEntity).ToList(),
            Total = total,
            Page = request.Page,
            Limit = request.Limit,
        };
    }
}
Multi-tenant filter is invisible: The handler calls _context.Products.Where(p => p.Name.Contains(search)) — it doesn't add .Where(p => p.CompanyId == _currentUser.CompanyId). The global query filter adds it automatically. CountAsync and the paginated ToListAsync both include the filter. If User A has 100 products and User B has 5, each sees their own count.

10. GetProductQuery, UpdateProductCommand, DeleteProductCommand

Three more files following the same pattern. GetProductQuery returns a single product by Id (scoped to company). UpdateProductCommand does a partial update — nullable fields, only non-null values are applied. DeleteProductCommand removes the product. All three query through _context.Products with the global filter active.

UpdateProductCommand — partial update pattern: Same as UpdateProfileCommand from Lesson 3. Every field is string? / decimal? / int?. Only non-null values are applied: if (request.Name is not null) product.Name = request.Name;. If the request changes the SKU, it checks for duplicates (excluding the product being updated).

5 endpoints: POST /products (201 Created), GET /products (paginated list), GET /products/{id} (detail), PUT /products/{id} (partial update), DELETE /products/{id} (204 No Content). All behind [Authorize] — unauthenticated requests get 401 before the controller runs.
What happens when a user creates a product
User A (from Company X) sends POST /products with a JWT containing companyId: "company-x". The controller maps the request body to a CreateProductCommand and calls _mediator.Send(). The handler checks that no other product in Company X has the same SKU (the AnyAsync is filtered by the global query filter). It creates new Product { Name = "Widget", SKU = "WGT-001", ... } — no CompanyId. SaveChangesAsync intercepts the new entity, sees CompanyId == Guid.Empty, sets it to _currentUser.CompanyId (= "company-x"), and inserts the row. The response returns a ProductDto with the generated Id. User B from Company Y can also create a product with SKU "WGT-001" — the composite unique index (CompanyId, SKU) allows it. User B will never see User A's products, and vice versa.
File chain: ProductsController.cs → CreateProductCommand.cs → CreateProductCommandHandler.cs → IApplicationDbContext.cs → AppDbContext.cs (query filter + auto-CompanyId) → PostgreSQL
